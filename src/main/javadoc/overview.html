Overview of Fortress Demo # 2 End-to-End Web Security

The purpose of this tutorial is demonstration of adequate security measures inside a Java Web environment. Security concerns covered over this tutorial are Authentication, Authorization, Confidentiality and Audit. Both declarative and programmatic enforcement controls will be used.

Security Mechanisms in the Tutorial

This sample Web application demonstrates the following policy enforcement techniques:
  1. Java EE Authentication with UserId and Password
  2. Java EE Coarse-grained Authorization using Roles
  3. Spring Coarse-grained Page Authorization using Roles
  4. Medium-grained Authorization inside Wicket Web Framework using RBAC Roles and Permissions
  5. Fine-grained Authorization inside Mybatis DAO using RBAC Permissions
  6. HTTPS Confidentiality
  7. LDAP v3 Confidentiality
  8. JDBC Confidentiality
  9. Audit Trail of Security Events

Prerequisites for Installation of Tutorial

  1. Debian or Redhat Liunx machine with OpenSSL installed
  2. Java 7 (or greater) SDK
  3. Apache Ant 1.8 (or greater)
  4. Apache Maven
  5. Firefox Web Browser
Infrastructure Installed during Tutorial
  1. OpenLDAP Directory Server
  2. OpenLDAP Fortress RBAC SDK
  3. MySQL Database Manager
  4. Tomcat Servlet Container
The techniques demonstrated in this demo are applicable to other Java Web application infrastructure configurations. If you find security flaws, or have questions about Fortress, submit to OpenLDAP Fortress Discussion list.
