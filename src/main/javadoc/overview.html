Overview of Fortress Demo # 2 End-to-End Web Security

The purpose of this application is to demonstrate correct security techniques inside a Java Web environment. Topics covered include Authentication, Authorization, Confidentiality and Audit. Both declarative and programmatic controls will be covered. The application installation requires completion of the sections listed below.

Sections required for installation:

  1. Heartbleed Bug
  2. Managing PKI Keys
  3. Set Hostname Entry
  4. Install Fortress
  5. Install MySQL
  6. Install Tomcat
  7. Install Commander RBAC Web Admin
  8. Install Fortress Demo #2

Security Mechanisms in the Tutorial

This sample Web application demonstrates usage of the following security mechanisms:
  1. Java 2 Authentication
  2. Java 2 Coarse-grained Role Authorization
  3. Spring Coarse-grained Page Role Authorization
  4. Medium-grained RBAC Authorization inside Wicket Web Framework
  5. Fine-grained RBAC Authorization within Data Persistence Classes
  6. Java 2 Confidentiality
  7. LDAP v3 Confidentiality
  8. JDBC Confidentiality
  9. Audit Trail of Authentication & Authorization Events

Prerequisites

  1. Debian or Redhat Liunx machine with OpenSSL installed
  2. Java 7 (or greater) SDK
  3. Apache Ant 1.8 (or greater)
  4. Apache Maven
Infrastructure Installed during Tutorial
  1. Fortress RBAC SDK
  2. OpenLDAP Directory Server
  3. MySQL Database Manager
  4. Tomcat Servlet Container
  5. Commander RBAC Web Admin
The techniques demonstrated in this demo are applicable to other Java Web application infrastructure configurations. If you find security flaws, or have questions about Fortress, submit to OpenLDAP Fortress Discussion list.
