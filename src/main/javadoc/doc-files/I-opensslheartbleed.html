Section I - Heartbleed Bug



This section provides instructions to fix the OpenSSL Heartbleed bug. Follow these steps if OpenSSL installed to target machine is version 1.0.1 through 1.0.1f. OpenSSL is used by OpenLDAP & MySQL for encryption of network traffic.

To determine if upgrade is needed, enter the following at the command prompt: 
    

# openssl version -b
If the date is less than Mon Apr 7 2014, and the version equals 1.0.1, then the machine is running an OpenSSL version that is vulnerable to the Heart Bleed bug and must be upgraded.

Steps to upgrade OpenSSL

  1. If Debian, you may need to update package lists in repo to latest: 
                
    
# sudo apt-get update
    If Redhat, to do similar: 
                
    
# yum clean headers && yum clean metadata && yum makecache
  2. To update openssl package to latest: 
                
    
# sudo apt-get upgrade openssl
    If Redhat: 
                
    
# yum upgrade openssl
  3. Verify that it worked (date will vary by linux distro): 
                
    
# openssl version -b
built on: Fri Jun 20 18:54:15 UTC 2014
